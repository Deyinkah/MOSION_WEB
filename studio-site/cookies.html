Cookies

Updated April 5, 2026

MOSION Studio uses limited first-party browser storage on studio.mosion.app to support sign-in continuity and notice state. No third-party advertising or cross-site tracking cookies are used.

01 Storage In Use

Cookies and browser storage on studio.mosion.app

The following first-party items are currently in use on studio.mosion.app.

Cookie: studio_known_user

Set after a Studio sign-in has been restored or completed successfully. Used to remember that this browser has signed in before, so Studio can recognise a returning browser without storing full account data in the landing page itself.

Type: first-party cookie. Retention: up to 12 months. Purpose: returning-browser recognition for Studio sign-in convenience.

Cookie / local storage: mosion_studio_cookie_notice_seen_v1

Used on the Studio landing page to remember that you dismissed the Studio cookie information banner, so the same message is not shown repeatedly after you have already seen it.

Type: first-party cookie plus local-storage fallback. Retention: up to 12 months. Purpose: banner dismissal state.

Local storage: mosion_auth_session_v1

Used by the Studio web app after sign-in to keep the active Studio session on the current browser. Clearing browser storage or signing out removes that remembered session.

Type: local storage. Retention: until sign-out or storage is cleared. Purpose: session continuity on this device.

02 If Cookies Are Blocked

If cookies are blocked

The Studio landing page can still load if cookies are blocked, but some convenience behaviour may stop working. Notice dismissal state may not be remembered, and returning Studio users may stay on the landing page instead of being sent straight to sign in.

Disabling cookies does not prevent the Studio landing page from loading. It mainly removes the browser memory layer that helps Studio recognise a returning browser for faster sign-in routing.
