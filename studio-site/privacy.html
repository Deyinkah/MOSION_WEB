Privacy & Cookies

How MOSION Studio uses browser storage.

Studio uses a small amount of first-party browser storage to keep sign-in smoother, avoid repeating the same notices, and remember the current Studio session on this device. We do not use the Studio storage listed below for advertising or third-party cross-site tracking.

Storage currently in use

These are the browser-level items currently used by the Studio landing page and Studio web app on the studio.mosion.app experience.

Cookie: studio_known_user

Set after a Studio sign-in has been restored or completed successfully. It tells the Studio landing page that this browser has signed in before, so returning users can be sent straight to the sign-in flow instead of always seeing the landing page first.

Type: first-party cookie. Retention: up to 12 months. Purpose: returning-user routing only.

Cookie/local storage: mosion_prototype_notice_seen_v1

Used on the Studio landing page to remember that the catalog preview notice has already been shown on this browser, so the same notice is not repeated on every visit.

Type: first-party cookie plus local storage fallback. Retention: up to 12 months. Purpose: notice frequency control.

Cookie/local storage: mosion_studio_cookie_notice_seen_v1

Used to remember that you dismissed the Studio cookie information banner, so we do not keep showing the same message after you have already seen it.

Type: first-party cookie plus local storage fallback. Retention: up to 12 months. Purpose: banner dismissal state.

Local storage: mosion_auth_session_v1

Used by the Studio web app after sign-in to keep the current Studio session on this browser. Clearing browser storage or signing out removes this stored session.

Type: local storage. Retention: until sign-out or browser storage is cleared. Purpose: session continuity on this device.

If cookies are blocked

The Studio landing page will still load, but some convenience behavior may stop working. In particular, returning users may stay on the landing page instead of being sent directly to sign in, and informational notices may reappear because the browser cannot remember prior dismissal state.

Terms
